# Explainability

Explainability means Bob can show how an answer was formed.

It is not a technical extra. It is the basis for trust.

## Practical Definition

An explainable Bob answer shows:

- the facts used
- the assumptions made
- source provenance
- confidence labels
- unresolved uncertainty
- correction history where relevant
- suggested next steps tied to evidence

## Why It Matters

Operational teams act on answers. If Bob says an install is blocked, a person may chase finance, contact a customer or update CRM.

That answer must be inspectable before it becomes operational action.

## Required Behaviour

Bob should:

- cite sources for important claims
- label inferred knowledge
- show conflicts rather than hide them
- state confidence plainly
- preserve corrections
- make audit history available

## Anti-patterns

- "because Bob says so"
- confidence with no source trail
- hiding conflicts to make the answer shorter
- treating memory as invisible truth
- turning assumptions into facts

## Alignment

Explainability supports:

- Discovery Lifecycle: sources are found before they are used
- Provenance and Trust Model: claims carry provenance and confidence
- Operational Memory Lifecycle: memory remains inspectable and correctable
- Query and Answer Lifecycle: every important answer can answer "why?"
