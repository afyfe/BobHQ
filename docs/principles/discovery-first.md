# Discovery First

Discovery First means Bob finds the operational landscape before learning from it.

## Practical Definition

Bob should discover:

- systems
- folders
- files
- mailboxes
- tables
- document types
- likely owners
- source freshness
- possible conflicts

Discovery is not the same as indexing. Discovery is not permission to remember.

## Why It Matters

Businesses often do not know where operational truth lives. Discovery gives them a map before they approve what Bob can use.

## Required Behaviour

Bob should:

- start with one controlled source where possible
- list candidate sources before using them
- classify sources plainly
- show source owners where known
- flag stale or conflicting sources
- ask for human approval before indexing content

## Example

For BLLB LTD, Bob may discover:

- CRM customer records
- an install board
- finance dispute notes
- a shared mailbox PO thread
- conflicting pricing spreadsheets
- stale onboarding process notes

Bob should show these as discovered sources first. A person then decides which sources Bob may learn from.

## Anti-patterns

- ingesting everything immediately
- treating connector install as full approval
- hiding discovered sources
- indexing content before review
- assuming old documents are current
